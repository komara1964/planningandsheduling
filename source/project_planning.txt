Обзор методологии разработки
----------------------------

Для управлением проектом будет использоваться Feature-Driven Development (FDD) процесс.
FDD является одной из гибких методик разработки программного обеспечения.
Основные шаги процесса:

* Построение объектной модели
* Построение списка функций (Feature)
* Планирование разработки по функциям
* Разработка по функциям. Шаги разработки:

  * Краткий обзор функции
  * Разработка проектной документации для функции
  * Разработка программного кода
  * Инспекция кода
  * Тестирование кода: автоматические тесты, функциональное тестирование
  * Выполнение построения программного кода и внедрения в производственную версию

Для контроля процесса будет вестись план работ на еженедельной основе.
